E. ## ISSUE 3

**Title:** Add a realistic, irregular 3D snowfall overlay for the first 18 seconds of the site

### Context
This site is built on Three.js (forked from bruno-simon.com). Add a new decorative snowfall effect that overlays the existing 3D scene/UI for the first 18 seconds after the site loads, then fades out. This must look organic and irregular — explicitly **not** a uniform grid of identical falling white boxes/squares moving in lockstep, which is the cliché, low-effort version of this effect to avoid.

### Requirements

**Visual irregularity (shape):**
- Each snowflake should be a small, low-poly, irregularly-shaped 3D object (not a perfect sphere or cube) — e.g. randomly scaled/non-uniformly stretched icosahedrons or similar low-poly geometry, varied per instance so flakes don't look identical to each other.
- Vary size noticeably across flakes (some smaller, some larger, within a sensible range).
- Use an efficient rendering approach given this is already an asset-heavy page — e.g. `InstancedMesh` with a capped particle count, rather than spawning hundreds of individual mesh objects.

**Movement irregularity:**
- Randomize each flake's: starting position, fall speed, rotation speed/axis, and horizontal drift/sway (e.g. a sine-wave sway with randomized amplitude and phase per flake) so no two flakes move identically and nothing reads as a repeating grid pattern.
- Stagger spawn timing slightly so flakes don't all appear at once.

**Overlay behavior:**
- The effect must render on top of the existing 3D scene/content as a visual overlay, without obscuring the view of what's underneath and without blocking any mouse/touch interaction with the existing site (driving the car, clicking UI, etc.). If implemented as a separate canvas/DOM layer, use `pointer-events: none`; if integrated into the main scene, make sure it's excluded from raycasting/click targets.

**Lifecycle:**
- The snowfall should run for the first 18 seconds (tie this to whatever you judge the most sensible start point — e.g. when the site finishes its initial load/the loading popup is dismissed — and note in the PR description which trigger point you chose).
- After 18 seconds, fade the snowfall out gradually (opacity transition over a couple of seconds) rather than stopping abruptly.
- Once fully faded, remove the snow objects from the scene and dispose of their geometries/materials so no memory or performance cost lingers afterward.

**Performance:**
- Keep particle count and geometry complexity conservative given the page is already 3D/WebGL-heavy. Reuse the existing render loop rather than spinning up a second one if at all possible.

### Acceptance criteria
- [ ] Snowfall looks organic/irregular in shape, size, and motion — no visibly repeating grid pattern or identical flakes.
- [ ] Effect overlays the scene without blocking clicks/drag/drive interactions.
- [ ] Effect starts on load, runs ~18 seconds, fades out smoothly, and fully cleans up (no leftover DOM nodes, geometries, or materials).
- [ ] No noticeable frame-rate drop or added jank on an already asset-heavy page.
- [ ] PR description states which load event was used as the 18-second start trigger.
